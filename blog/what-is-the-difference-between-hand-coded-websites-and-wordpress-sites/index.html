Quelle est la différence entre les sites Web codés à la main et les sites Wordpress ?

Ryan Postell 7 mars 2021

Sites Web statiques (codés à la main) vs sites Web dynamiques (WordPress, Wix, etc.)

Points clés à retenir :

  • Les sites Web statiques codés à partir de zéro se chargent jusqu’à 10 fois plus rapidement que les constructeurs de pages comme WordPress. Si votre site Web met plus de 3 secondes à se charger, la majorité de vos visiteurs abandonneront le site, perdant ainsi des clients.

  • Google indexe votre site Web en fonction des performances de votre site mobile, et les sites mobiles de création de pages ne sont pas aussi optimisés, pas aussi rapides, pas aussi bien construits et souvent pas 100 % réactifs, ce qui pourrait nuire à votre classement dans les recherches.

  • Mes sites statiques créés à la main sont d'abord codés pour les mobiles, ce qui signifie que j'écris le code en commençant par les écrans mobiles, puis en ajoutant du code pour les tablettes et les ordinateurs de bureau. Cela rend le mobile plus réactif, optimisé et incroyablement rapide à charger, améliorant ainsi les performances de votre site Web sur Google et garantissant que les visiteurs n'abandonnent pas votre site.

  • WordPress est notoirement connu pour ses vulnérabilités et doit être mis à jour régulièrement pour corriger les failles de sécurité que les pirates peuvent utiliser pour compromettre votre site Web. Les sites statiques créés à la main ne peuvent pas être compromis aussi facilement et sont pratiquement impénétrables en raison de leur simplicité et de l’absence de points d’entrée à pirater.

Dans le monde du développement Web, il existe deux types de sites Web : statiques et dynamiques. UN site statique est un site qui ne contient que les principaux éléments constitutifs du développement Web : HTML, CSS et JavaScript. Il n'y a pas de bases de données, de fonctionnalités, de manipulation de données ou quoi que ce soit de sophistiqué. Il s’agit simplement d’afficher la page telle qu’elle a été créée. Ces trois langages de codage constituent tout ce que vous pouvez voir sur un site Web et, dans de nombreux cas, c'est tout ce dont vous avez besoin. Lorsque vous chargez un site statique, c'est votre navigateur qui lit et compile le code du site Web et le charge instantanément. Chaque utilisateur charge exactement le même contenu. Et l’une des meilleures façons de créer un site statique est de le coder à la main (c’est ce que je fais de mieux !) ou avec un générateur de site statique.

Un site dynamique (WordPress, constructeurs de pages) utilise des langages et des technologies supplémentaires côté serveur pour extraire le HTML, le CSS et les ressources d'une base de données et assembler le site Web au fur et à mesure qu'il se charge sur le serveur, puis il est envoyé au navigateur. Au lieu que les navigateurs compilent le code, c'est le serveur qui le compile pour le navigateur et l'envoie, et au lieu que chaque utilisateur reçoive exactement le même contenu, ils reçoivent tous un contenu différent individuellement. Cela ralentit les temps de chargement et votre site Web est vulnérable aux attaques de logiciels malveillants et de pirates informatiques interrompant le processus de vérification par le serveur de la base de données pour le contenu du site Web. Les sites statiques n’ont pas de bases de données, il n’y a donc rien à attaquer. Nous y reviendrons davantage plus tard dans la page.

Les sites dynamiques incluent des sites Web créés par WordPress, Wix, Squarespace, GoDaddy, un constructeur de pages ou construits en PHP, etc. Ces constructeurs de pages stockent les actifs et le contenu dans une base de données et lorsque quelqu'un visite le site Web, le serveur vérifie son inventaire des éléments demandés par le navigateur et les extrait de cette base de données et les rassemble dans la page de destination que vous voyez. Bien que ces créateurs de pages facilitent la création de votre propre site Web et à moindre coût, ils présentent d'ÉNORMES inconvénients par rapport aux sites Web statiques que je crée.

Pourquoi les sites Web statiques sont-ils plus rapides que les sites Web dynamiques ?

L'un des aspects les plus importants du classement de votre site Web sur Google est la rapidité avec laquelle il se charge et fonctionne, en particulier sur mobile. Le problème avec les sites Web dynamiques (WordPress, Wix, etc.) est que le serveur met plus de temps à vérifier la base de données et à tout rassembler. Et plus vous avez de plug-ins et de contenus supplémentaires sur votre site, plus vous ne ferez que le ralentir. Ceci est essentiel car des études ont montré que la majorité des utilisateurs abandonneront un site Web si le chargement prend plus de 3 secondes. Donc, si votre site Web prend plus de 3 secondes à charger, à quoi bon l’avoir si la majorité des utilisateurs ne le voient même pas ? C'est l'un des LE PLUS GRAND inconvénients que je vois avec les sites WordPress et Wix : ils peuvent prendre trop de temps à charger et vous coûter de l'argent.

Les sites statiques, cependant, peuvent se charger jusqu'à 10 fois plus rapidement qu'un site Web dynamique, car ils n'ont pas besoin de procéder à une opération chirurgicale du site Web et de rattacher les membres de la page aussi rapidement que possible sur la table d'opération du navigateur. Tout est déjà prédéfini, il suffit donc au navigateur de le charger et de nous le lancer. Il n’y a pas de plugins, pas de bases de données, pas d’espace perdu ou de surcharge. Chaque ligne de code que j'écris sur mes sites est conçue dans un but précis et écrite de manière organisée, facile à lire et à comprendre pour Google. Les sites dynamiques comme WordPress contiennent des TONNES de code volumineux et désordonné qui ne font qu'enliser davantage les temps de chargement.

Imaginez si vous avez une assiette de spaghettis et que vous voulez savoir combien de nouilles il y a dans votre assiette. Eh bien, ils sont tous mélangés et emmêlés les uns dans les autres, il faudra beaucoup de temps pour les séparer, compter le nombre de nouilles qu'il y a et leur donner un sens. Dans votre autre assiette (votre assiette « statique »), vous avez toutes vos nouilles alignées en jolis groupes de 10 dans l'assiette avec la viande à côté, prêtes à manger (s'il vous plaît, ne mangez pas de spaghettis comme celui-ci). Il est beaucoup plus facile et rapide d’analyser le nombre de nouilles que vous avez lorsque tout est regroupé et organisé en conséquence. C'est la différence entre l'apparence du code d'un site dynamique et celui d'un site statique. C’est ce qu’on appelle le « code spaghetti ». Et cela rend le chargement d'un site plus long à analyser et à charger par le navigateur, et les robots de recherche de Google auront du mal à comprendre votre contenu et ce qu'il signifie car il est caché dans un fouillis de code sans contexte.

Le temps de chargement de référence pour un site Web est de 2 secondes. Mes sites Web statiques créés à la main peuvent souvent se charger en moins de 0,5 seconde, ce qui est nettement inférieur à la moyenne. Ils se chargent presque instantanément. Le code est regroupé et organisé afin que sa lecture et sa compréhension soient plus faciles pour votre navigateur et pour Google, ce qui améliore les temps de chargement et empêche les clients de quitter votre site et améliore le classement de vos sites Web sur Google. Donc, si votre site Web ne génère pas beaucoup d’affaires, vérifiez combien de temps il faut pour charger cet outil à partir de Google.

Si le temps de chargement dépasse 3 secondes, il y a de fortes chances que les gens partent avant même de l'avoir atteint.

En parlant de Google et de vitesse, ils ont récemment mis à jour leur algorithme de recherche dans lequel ils classent votre site Web en fonction des performances du site mobile, appelé Mobile First Indexing. Donc, fondamentalement, plus votre site mobile se charge rapidement, meilleur est votre classement. C’est là que les sites statiques gagnent à chaque fois.

Programmation mobile d’abord et vitesse accrue du site

La programmation mobile first est quelque chose qui ne peut pas être réalisé par un constructeur de pages. Il s'agit d'un modèle de conception dans lequel nous, en tant que développeurs, écrivons le code en commençant par les écrans mobiles et en ajoutant du code pour les tablettes et les ordinateurs de bureau au fur et à mesure que nous parcourons le fichier. Lorsque le navigateur lit le fichier CSS pour voir comment la page doit être structurée et stylée, il le lit de haut en bas. Si nous commençons par écrire du code de bureau en haut, le navigateur le lira d'abord et le chargera, puis il continuera à parcourir la page en vérifiant les règles pour différentes tailles d'écran et en chargeant ces conceptions en modifiant le contenu du bureau pour l'adapter au mobile. Si vous êtes sur mobile et que le code mobile se trouve au bas du fichier CSS, le navigateur doit lire TOUT le code de ce fichier, charger le code du bureau, vérifier la règle de taille d'écran et dire "oui, c'est bien mobile". Il doit ensuite tout décomposer depuis le bureau et tout intégrer dans l'écran du mobile. Cela fait perdre du temps et des ressources.

Avec l’approche mobile first, votre code mobile est en haut et est lu en premier. Ainsi, lorsque vous êtes sur votre téléphone et que le site se charge, il charge d'abord le code de style mobile, vérifie les règles de taille de l'écran et dit "Eh bien, c'est tout les garçons, affichez ce code à l'écran. C'est l'heure du spectacle !".

Laissez-moi vous dire par expérience : il est plus facile de faire grandir le contenu dans son espace au lieu de le réduire et de l'entasser dans des espaces plus petits. La programmation mobile first + le développement de sites statiques rendront votre site ultra-rapide. Cela pourrait même choquer certaines personnes.

Pourquoi les sites statiques sont-ils plus sécurisés que les sites dynamiques ?

Les sites dynamiques disposent de bases de données et de plates-formes côté serveur qui peuvent être attaquées lorsqu'ils tentent d'accéder à la base de données ou aux systèmes de gestion de contenu qui stockent toutes les données de votre site Web. Ainsi, pendant que le serveur construit le site et le prépare pour le navigateur, les pirates peuvent s'infiltrer dans le processus et compromettre votre site Web.

Les sites statiques n’ont cependant rien à attaquer. Il n'y a pas de base de données, pas de jetons de session, aucune information collectée ou transmise pour qu'un pirate informatique puisse interférer ou intercepter. Il n'y a aucune étape pour qu'ils se glissent et glissent quelque chose dans la poche arrière de notre site Web avant de parvenir au navigateur.

L’un des problèmes des sites WordPress est la quantité de plugins que vous devez utiliser. Si vous créez un site WordPress et que vous l’oubliez pendant un moment, beaucoup de ces plugins auront été mis à jour pour corriger les vulnérabilités de sécurité et si vous ne les mettez pas à jour, votre site ne sera pas à l’abri des attaquants et parfois les plugins obsolètes ne fonctionneront tout simplement pas bien avec les autres et supprimeront tout le site sans que vous le sachiez. Qui a le temps de suivre toutes ces mises à jour et correctifs ? C’est là que mon travail de création de sites statiques se démarque vraiment : aucune mise à jour n’est nécessaire. C'est juste du HTML et du CSS. Il n’y a aucun correctif de sécurité à appliquer manuellement pour assurer la sécurité de votre site. Quand c’est fini, c’est fait.

Pensez-y comme ceci : vous avez deux cabanes dans les bois, l'une a l'électricité et Internet et des serrures électroniques sophistiquées sur les portes que vous contrôlez avec une application et les lumières et les appareils sont connectés à Internet et au cloud que vous contrôlez depuis votre téléphone. C'est vivre ! Dans l’autre cabine, elle n’est connectée ni à un réseau ni à un nuage, la cheminée produit toute la chaleur et les serrures des portes ne sont que de vieux pênes dormants.

Eh bien, dans l'autre cabane chic, quelqu'un a piraté leur connexion Internet, a pris le contrôle des serrures de la maison et les a désactivées, leur permettant ainsi d'entrer et de tout prendre. Toutes ces cloches et sifflets sophistiqués ont simplement créé davantage de points d'entrée permettant aux attaquants d'entrer dans votre cabine alors que tout ce dont vous aviez besoin était de vous détendre après une randonnée.

Mais l’autre cabine, qui n’avait pas d’électricité ni de serrures ni de dispositifs contrôlés par Internet, est toujours sûre. Ils se débrouillent avec l’essentiel comme un poêle à feu et des lanternes. C'est du HTML et du CSS. Juste de bons vieux poêles et lanternes. Et les serrures de la porte sont à l’abri des attaques. Personne n’y entre. C’est ce qu’est un site statique. Il supprime tous les extras et n’est qu’un site Web construit à partir de ses éléments de base. C’est le type de site Web que je crée. Ce sont des cabanes en rondins coupées et construites à la main avec uniquement les outils d’un artisan. Ces constructeurs de pages sont produits en masse et présentent des risques de sécurité de moindre qualité.

Et comme WordPress est le système de gestion de contenu (CMS) le plus populaire pour la création de sites Web, les pirates cibleront spécifiquement les sites WordPress pour exploiter les vulnérabilités connues et cibleront les sites Web dont les versions et les plugins sont obsolètes. Votre site Web a une cible au dos. En empruntant la voie du site statique, nous supprimons cette cible de votre dos et votre site Web peut vivre librement sans craindre d'être enlevé dans les rues d'Internet.

Mais qu’en est-il des serveurs sur lesquels vivent ces sites statiques ? Ne peuvent-ils pas être compromis et le site peut-il tomber en panne ? Non! Lorsque votre site statique est desservi par un réseau de diffusion de contenu (CDN), votre site réside sur plusieurs serveurs à travers le monde, prêts à être consultés. Ainsi, lorsqu'un serveur tombe en panne, il peut simplement être chargé à partir d'un autre serveur situé à un emplacement différent. Pas de temps d'arrêt !

Sites statiques faits à la main vs WordPress et Wix

Dans l’ensemble, les sites Web statiques créés à la main battront presque TOUJOURS les constructeurs de pages comme WordPress et Wix. Les sites Web statiques se chargent en millisecondes, alors que cela peut prendre plusieurs secondes ou plus pour un site WordPress ou Wix. Et le fait qu’ils soient presque pratiquement impénétrables contre le piratage ou la compromission en raison de leur simplicité en fait le gagnant incontesté dans la lutte pour votre entreprise.

Les sites dynamiques n'offrent actuellement aucun moyen de faire de la programmation mobile first et ne sont pas en mesure de profiter de ses avantages. L'un des problèmes que vous pouvez constater par vous-même est de trouver un site Wix et d'essayer de redimensionner le navigateur. Vous remarquerez qu'ils ne proposent pas de version pour tablette de votre site et que la page entière n'est pas très réactive, ce qui signifie que le contenu ne change pas ou ne bouge pas à mesure que l'écran s'agrandit ou se rétrécit. Sur certaines tailles d'écran, le contenu de votre page Web est coincé dans sa boîte et est en fait coupé sur les bords de l'écran car il n'a nulle part où aller et maintenant les gens ne peuvent même plus lire ce qui s'y trouve. Ou encore, le texte peut commencer à s'enrouler autour des images de manière étrange, ce qui déforme la mise en page et donne l'impression que votre site est cassé. Ces constructeurs de pages ne vous donnent pas le contrôle dont vous avez besoin pour rendre votre site réactif à n'importe quelle taille d'écran possible sans savoir comment coder et accéder à l'API des constructeurs de pages pour communiquer avec leur système et effectuer un travail très personnalisé et compliqué.

Pourquoi devrais-je payer quelqu’un pour créer un site Web alors que je peux le créer moi-même ?

Wix et WordPress annoncent que n'importe qui peut créer un site Web gratuitement, ce qui le rend très attrayant pour les petites entreprises qui n'ont pas l'argent nécessaire pour embaucher un développeur comme moi pour en créer un. Cependant, si la vitesse des pages, les temps de chargement, la sécurité, la conversion d'un plus grand nombre de clients et le classement Google sont des préoccupations que vous aimeriez voir abordées dans un site Web et que vous souhaitez vous démarquer de la concurrence, je vous conseille fortement de trouver quelqu'un pour créer un site Web statique pour vous (comme moi !). Pensez-y de cette façon : n’importe qui peut simplement acheter de la peinture et peindre sa propre maison. Mais tout le monde ne pourra pas le faire avec la même qualité qu’un professionnel, et leur maison en souffrira et aura un aspect horrible. Tout comme la peinture, le développement Web est un métier et peut coûter un peu cher. Avoir un site statique construit par un développeur peut coûter entre 1 800 $ et plus de 3 000 $ selon la complexité. Il est donc naturel d’être attiré par les créateurs de pages et d’économiser de l’argent en le faisant vous-même. Il n’y a rien de mal à ça ! Mais si vous souhaitez faire passer votre entreprise à l’étape suivante, arrêtez de laisser de l’argent sur la table et envisagez de créer un site statique. Créez quelque chose qui fera plus de kilomètres et devancez la concurrence.

Et certains développeurs créent des sites WordPress personnalisés et font du bon travail et se soucient de ce qu’ils font. Je ne dis pas que TOUS les sites Web ou les développeurs WordPress sont des escrocs ou fabriquent des produits épouvantables. Ceux-ci sont toujours tout à fait acceptables et ont leur utilité. Si vous avez besoin d'un site Web qui DOIT disposer d'une base de données et créer dynamiquement des centaines de pages en fonction du contenu demandé par le visiteur, c'est là que WordPress brille. WordPress est idéal pour les personnes qui souhaitent modifier leur propre contenu, créer une boutique en ligne, un blog qu'elles peuvent écrire et modifier elles-mêmes, des applications personnalisées ou manipuler des données, et tout ce qui nécessite une base de données. Les sites statiques ne sont que cela : ils sont statiques, immuables. Ils sont parfaits pour les sites Web qui ont uniquement besoin d’afficher des informations. Ils ne le stockent pas, ne le manipulent pas et ne génèrent pas de nouveau contenu individuellement pour chaque visiteur. Utiliser Wordpress pour créer un site strictement informatif, c'est comme utiliser un marteau-piqueur pour enfoncer un clou.

Il y a de fortes chances que vos concurrents disposent également d’un site WordPress. Certains développeurs ou agences facturent des milliers de dollars pour créer ces sites WordPress simples qui auraient pu être rendus statiques, et beaucoup d'entre eux ne sont que des modèles et des dumps qu'ils vous remettent sans support, wham bam merci madame. Il sera donc difficile de trouver un bon développeur ou une bonne agence, mais cela en vaudra la peine. Il vous suffit de prendre le temps d’éliminer les mauvais des bons.

Investissez dans votre site Web

D'après mon expérience, il existe deux types de personnes : les propriétaires d'entreprise et les entrepreneurs. L’une des erreurs les plus courantes commises par les propriétaires d’entreprise lorsqu’il s’agit d’un site Web est qu’ils le considèrent comme une dépense plutôt qu’un investissement. Les entrepreneurs comprennent que lorsque vous dépensez de l’argent pour votre entreprise, c’est un investissement. INTO l'entreprise, pas une dépense, et peut en justifier la valeur en voyant le retour que l'investissement apportera. Par exemple, vous achetez un site Web à un développeur pour 3 000 $ afin de créer un site statique pour vous. Cela semble beaucoup, mais avec un site plus beau et plus performant, vous pourrez peut-être convertir les visiteurs en clients qui auraient quitté le site parce que le chargement a pris trop de temps ou qu'il avait l'air obsolète et horrible et comme si on ne pouvait pas vous faire confiance. 3 000 $ rapportent soudainement 10 000 $ de plus par an ou plus après un certain temps dans de nouvelles affaires. Si vous pouviez dépenser 3 000 $ et gagner 10 000 $ en un an, feriez-vous cet « investissement » ? Je sais que j'ai sorti ces chiffres de nulle part, mais cela ne veut pas dire que ce n'est pas possible. Certains clients ont doublé leur chiffre d'affaires après un an de travail avec moi.

Voyons comment je fais les choses. Je vends des sites Web à 175 $ par mois. Certains pourraient se demander « pourquoi devrais-je payer chaque mois pendant des années ? » Pensez-y, avec tous les avantages de mes sites statiques, est-ce que cela a rapporté plus de 175 $ par mois ce mois-ci ? Les réponses seront presque toujours oui. Un peintre, par exemple, gagnerait plus de 800 $ par nouveau client. Si mon site n'apporte qu'un seul client supplémentaire par mois, cela représente un retour de 650 $. Et chaque nouveau client que vous recevez chaque mois ne fait qu'ajouter à la valeur que vous recevez de mes sites. Et pour ces 175 $ par mois, vous avez quelqu'un de garde 24h/24 et 7j/7 pour gérer votre site Web à votre place, mettre à jour le site en cas de changement dans les priorités des algorithmes de recherche et répondre à toutes vos questions. Vous obtenez plus qu'un simple site Web, vous obtenez un partenaire qui s'investit également dans le succès de votre entreprise.

Les sites Web ressemblent beaucoup aux tatouages : vous en avez pour votre argent. Si vous voulez quelque chose qui semble générique ou obsolète et qui pourrait inciter les gens à vous éviter, alors c'est ce que vous obtiendrez pour un site Web à 300 $ de Craigslist ou Fiverr. Tout comme pour trouver un tatoueur, vous devez faire des recherches et trouver des développeurs réputés qui peuvent répondre à toutes vos questions et simplement créer un beau produit de haute qualité.

Ainsi, avec toutes ces options rapides et peu coûteuses disponibles pour créer un site Web, il est important de savoir qu’un site Web de qualité ne viendra jamais à ce niveau de prix. Cela devrait en fait être dissuasif. Si je devais me faire opérer, je ne ferais certainement pas confiance au médecin que j’ai trouvé en ligne pour le faire pour environ 350 $ alors que cela coûte 4 000 $ partout ailleurs. Cela crie simplement au manque de qualité et au manque d'effort et quelque part le long de la ligne, ils sautent quelques étapes pour le lancer le plus rapidement possible afin de gagner autant d'argent que possible avec le moins d'effort possible. Votre entreprise mérite mieux. Si vous voulez un site bon marché, attendez-vous à des résultats bon marché. Mais si vous voulez que tout soit bien fait, c'est là qu'intervient un développeur.

Conclusion:

En fin de compte, que vous optiez pour un site statique ou un site dynamique, tout dépend de votre budget et de vos priorités. Si vous n’avez pas beaucoup à dépenser pour un site Web et que vous avez juste besoin de quelque chose, vous pouvez utiliser WordPress ou Wix. Mais si vous avez le budget nécessaire pour réaliser quelque chose qui représente un pas en avant et peut maximiser votre présence en ligne, alors créer un site statique est votre meilleur retour sur investissement. Ou si vous avez besoin de quelque chose de plus grand qui implique la nécessité d'une base de données ou la possibilité de modifier votre contenu vous-même, alors WordPress et d'autres créateurs de pages sont la voie à suivre.

Avec mon modèle commercial, vous n’avez pas à choisir entre le faire vous-même pour économiser de l’argent ou demander à un développeur de le faire à un prix élevé. Je suis capable de faire ce genre de travail pour si peu d’avance parce que je suis un père au foyer – mes factures sont déjà payées, je dois juste rester marié ! Je peux donc me permettre d'investir tout ce travail sur mes sites pour si peu d'avance, car je n'ai pas de frais d'exploitation, ni de bureau ou d'employés. C'est juste moi.
